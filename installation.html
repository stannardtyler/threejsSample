Threejs Sample

Welcome to this simple example of three js. Basically the three js library is being loaded from the web rather than local library files.

Loading the library files this way is beneficial in two ways as it speeds up the load time of the website and can run locally as well. However, not having an internet connection you won't be able to make any edits. In this case you'll want to download the libraries from the threejs website

The first link is a sample file that contains an html file with a script syntax that loads the libraries. Next, the js file with the same name loads these libraries at the top.

sample with URL files

Adding additional addons and features of the threejs library will require additional loaders and libaries. For this, you'll need to follow the same directory flow as if you are loading the files locally. Review this file to see how the gltfloader addon is being called. This "directory" flow is defined in the html page of the same name in the script syntax, same as in the sample file.

So to load the addon for the gltf-loader, which is visible here as an live site example // github repo code, we follow the same directory but place it within the js file. Example:

This is how the addon javascript files are loaded on the html page ->>
"three/addons/": "https://unpkg.com/three/examples/jsm/"

The gltf-loader has the following directory path ->>
three.js/examples /webgl_loader_gltf.html

So we write it the following in the three js file:

import { GLTFLoader } from "three/addons/loaders/GLTFLoader.js";

This is the same way the examples are loaded. I'd recommend you to attempt to load the examples in with this start repo as separate html and javascript files and link them in a similar way. Make multiple pages with different names to keep previous versions as prototypes.
